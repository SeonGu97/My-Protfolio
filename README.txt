- creator 리펙토링 완성.

- 약 2주 동안, creator를 리펙토링했다.
이전의 creator는 인스턴스를 만들고 그 인스턴스의 엘리먼트를 만드는데 코드가 이러했다.

" const name = new creator(name, element, type, value, text, num, parent, boolean, event, func); "

이런식으로, 너무 많은 데이터를 직접 넣어야했고 만약 엘리먼트가 여러개라면 document.querySelectorAll을
또 작성해서 사용해야 됬었다. 불필요하게 코드가 길어졌고 변수를 다루기가 불편했는데 바뀐 코드를 보면

" loop(creator, data, parent); "

이게 다이다. 이렇게 간단한 코드가 가능한건 데이터를 따로 관리하기 때문이다. data.js를 만들어서

{
    name: 'index',
    element: 'div',
    type: ['id', 'class'],
    value: ['index', 'index'],
    text: ['true'],
    number: 1,
    array: [],
    parent: null,
}

요런식으로 만들고 싶은 인스턴스의 엘리먼트 정보를 입력하면 생성이된다. 이렇게 만들면 장점이 유지/보수가 편해진다.
정보가 한 곳에 있으니까 바로바로 바꾸면된다. 그리고 이전에 만약 여러 엘리먼트를 만들고, 그 엘리먼트중에 특정 엘리먼트에만
무언가를 하고싶다면 document.querySelectorAll 추가해서 해당 엘리먼트를 가져왔다면, 이번에는 엘리먼트를 만듦과 동시에
배열에 집어 넣어서, 여러개를 만든 엘리먼트들이 하나의 배열에 들어가있다. 그래서 모두 사용하고 싶으면 모두 사용해도 되고,
특정 엘리먼트만 사용하고 싶다면 뒤에 [0]이런 식으로 배열을 가져오면 된다. 이렇게 하면 selector를 사용할 필요가 없다.
그리고 아직 이벤트는 동적으로 추가가 안되는데, 이건 수동적으로 사용하는게 편할지 아니면 동적으로 하는게 편할지 아직 고민
중에 있다. 일단 엘리먼트를 추가하고 속성이나 텍스트등을 추가하는건 완성됬다. 일단 써보면서 수정해야겠다.
